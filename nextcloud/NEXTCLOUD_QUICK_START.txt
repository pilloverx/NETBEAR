TIPS & TRICKS
================================================================================

1. Start with Recon to understand the system
   ‚Üí Gives you user IDs, share structure, public links

2. Use results to inform other tests
   ‚Üí Known file IDs help with IDOR testing
   ‚Üí User enumeration helps with auth testing

3. Increase IDOR_SAMPLE_SIZE for larger instances
   ‚Üí Default 50 is fast; try 200-500 for thorough testing

4. If tests seem to timeout/fail:
   ‚Üí Increase RATE_LIMIT_SEC to 1-2 seconds
   ‚Üí Reduce MAX_WORKERS to 2-3
   ‚Üí Check network connectivity

5. For stealth/evasion:
   ‚Üí Increase rate limit to 2-3 seconds
   ‚Üí Use rotating proxies via config.py PROXIES list
   ‚Üí Reduce worker count to 1-2

TROUBLESHOOTING
================================================================================

Issue: "Connection failed"
  ‚Üí Verify host URL (https://...)
  ‚Üí Check username/password
  ‚Üí Try: python -c "import requests; requests.get('URL')"

Issue: "No findings despite vulnerabilities"
  ‚Üí Increase IDOR_SAMPLE_SIZE in config.py
  ‚Üí Decrease RATE_LIMIT_SEC to test faster
  ‚Üí Check if instance is heavily patched

Issue: "Tests timing out"
  ‚Üí Increase NEXTCLOUD_TIMEOUT in config.py
  ‚Üí Reduce MAX_WORKERS
  ‚Üí Increase RATE_LIMIT_SEC

Issue: "SSL certificate error"
  ‚Üí Set NEXTCLOUD_VERIFY_SSL = False in config.py
  ‚Üí Only use for self-signed certs in testing!
